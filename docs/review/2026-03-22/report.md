## P0 Findings

### P0-1. `has_signature` wrapper struct codegen generates unbound variable reference
**Crate:** kirin-derive-ir | **File:** `crates/kirin-derive-ir/src/has_signature.rs:18-28`
**Perspective:** Soundness Adversary

The `signature_body_struct` function for `#[wraps]` structs generates code referencing a field binding (e.g., `field_0`) without first destructuring `self`. The generated method body would be `<WrapperTy as HasSignature>::signature(field_0)` where `field_0` is unbound. Compare with `bool_property.rs:183-189` which correctly emits `let Self #pattern = self;`.

**Action:** Add destructuring `let Self #pattern = self;` before the delegation call in the wrapper branch, mirroring `BoolProperty::for_struct`.

## P1 Findings

### Core IR (kirin-ir) — 4 findings

#### P1-1. Statement::detach does not clear BlockInfo::terminator cache
**File:** `crates/kirin-ir/src/detach.rs:13`
After detaching a terminator statement, `BlockInfo::terminator` still points to the orphaned statement ID. Any subsequent query trusting the terminator cache observes stale data.
**Action:** In `Statement::detach`, check if `parent_info.terminator == Some(*self)` and clear to `None`.

#### P1-2. Linked list length underflow in release builds
**File:** `crates/kirin-ir/src/detach.rs:53-57`
Length decrement guarded only by `debug_assert`. In release builds, double-detach or stale parent causes `usize` wrapping to `usize::MAX`, corrupting all `ExactSizeIterator` impls.
**Action:** Replace `debug_assert` + raw decrement with `checked_sub(1).expect(...)`.

#### P1-3. Arena::gc() invalidates all external IDs with no safety mechanism
**File:** `crates/kirin-ir/src/arena/gc.rs:27`
`gc()` is `pub` on `Arena`. All previously obtained IDs become stale with no generation counter or runtime detection.
**Action:** Make `gc()` private/`pub(crate)` and only expose through a safe whole-stage compaction API, or add generation counters.

#### P1-4. DenseHint::insert_or_combine silently drops value when ID is out of range
**File:** `crates/kirin-ir/src/arena/hint/dense.rs:46-59`
`get_mut` returns `None` for out-of-range IDs and the value is silently dropped. `insert()` on the same type resizes dynamically — inconsistent behavior.
**Action:** Add resize logic from `insert()` to `insert_or_combine`, or return `Result`.

### Parser Runtime (kirin-chumsky) — 5 findings

#### P1-5. Graph emit error paths leak relaxed-dominance mode
**File:** `crates/kirin-chumsky/src/ast/graphs.rs:192-198, 298-309`
If `emit_statement` returns `Err` after `set_relaxed_dominance(true)`, cleanup is skipped. Subsequent SSA lookups silently create forward-reference placeholders instead of reporting `UndefinedSSA`.
**Action:** Use RAII scope guard pattern (`ctx.relaxed_dominance_scope()`) that restores state on drop.

#### P1-6. Region emit error path leaks scope
**File:** `crates/kirin-chumsky/src/ast/blocks.rs:249-281`
`push_scope()` at line 249 but `pop_scope()` at line 281 is skipped on `?` propagation. Same class of bug as P1-5.
**Action:** Use `ScopeGuard` or closure-based cleanup.

#### P1-7. `parse_text.rs` panics on `link()` failure
**File:** `crates/kirin-chumsky/src/function_text/parse_text.rs:377, 432`
`.expect("link should succeed...")` in a `Result`-returning parsing API.
**Action:** Replace with `.map_err(...)`.

#### P1-8. `collect_function_lookup` and `collect_staged_lookup` unwrap arena entries
**File:** `crates/kirin-chumsky/src/function_text/parse_text.rs:778, 819`
`info.clone().unwrap()` panics on tombstone/None entries.
**Action:** Use `filter_map` or handle None.

#### P1-9. `fn_symbol` panics on unnamed functions
**File:** `crates/kirin-chumsky/src/function_text/parse_text.rs:808-813`
`.expect("stage declarations should always use named functions")` in production code path.
**Action:** Return `Result<GlobalSymbol, FunctionParseError>`.

### Derive Infrastructure — 2 findings

**File:** `crates/kirin-derive-toolkit/src/template/builder_template/helpers.rs:559-587`
When a `#[wraps]` variant has extra fields, the `From` impl accepts `value` but never places it in the constructor — the wrapped value is silently discarded.

## Cross-Cutting Themes

### 1. Panics in public APIs that should return Results (6 reviewers, 6 crates)

The most pervasive issue. At least **15 of the 23 P1 findings** are panics (via `.expect()`, `.unwrap()`, or unchecked arithmetic) in public API paths that should return `Result`. This spans:
- Core IR: arena GC, detach, DenseHint
- Parser: link, collect_function_lookup, fn_symbol
- Interpreter: active_stage_info, expect_info, expect_stage_id
- Printer: bat::print_str
- Dialects: ForLoopValue::loop_step

**Pattern:** Internal code uses `expect()` for "should never happen" conditions, but these conditions ARE reachable through the public API with misconfigured inputs.

### 2. Scope/state leaks on error paths (Soundness, 1 crate)

Parser emit functions (`DiGraph::emit_with`, `UnGraph::emit_with`, `Region::emit_with`) set state flags or push scopes before emitting, but `?` propagation skips cleanup. All three need RAII guard patterns.

### 3. Missing `#[must_use]` (Code Quality, all crates)

Zero `#[must_use]` across most of the workspace. Key missing annotations:
- `Continuation<V, Ext>` (silently discarding control flow)
- `Arena::gc()` return value (must use IdMap)
- All error types (ParseError, EmitError, InterpreterError)
- Builder finalize methods
- FunctionRenderBuilder, PipelineRenderBuilder

### 4. SCF dialect expressiveness gaps (Formalism, 1 crate)

`scf.for` and `scf.if` both lack result values, making them control-flow-only constructs rather than the expression-oriented structured control flow they model in MLIR. This is a design gap that limits dialect utility.

### 5. Derive codegen correctness (Soundness, 2 crates)

Two bugs in derive codegen: P0-1 (unbound variable in has_signature) and P1-10 (value drop in from_impl). Both affect code generation for `#[wraps]` patterns.

### 6. Code duplication (Code Quality, 4 crates)

- Detach macro duplicates Statement::detach logic (~50 lines)
- BFS/graph-building between UnGraphBuilder and attach_nodes_to_ungraph (~120 lines)
- port_list/capture_list identical parsers
- UnitTy defined 6x across dialect test files
- Name resolution `^name` pattern 3x in printer

## Architectural Strengths

1. **Trait decomposition** — ValueStore / StageAccess / BlockEvaluator / Interpreter layering is clean and well-motivated
2. **Template system** — Input::compose().add(template).build() pipeline is composable and extensible
3. **Graph unification** — GraphInfo<L, D, Extra> shares code cleanly between directed/undirected graphs
4. **Error taxonomy** — Three distinct parser error types, 9-variant InterpreterError, structured StageResolutionError
5. **Lattice formalism** — Lattice / HasBottom / HasTop / FiniteLattice / TypeLattice hierarchy is clean
6. **Diagnostic attributes** — `#[diagnostic::on_unimplemented]` on key traits provides actionable error messages
7. **Monomorphic dispatch** — ParseDispatch eliminates HRTB, DynFrameDispatch uses efficient function pointers
8. **Prior review items addressed** — 4 of 5 printer issues fixed, graph duplication unified, SSA resolution centralized
