# Kirin RFC Checklist

Use this checklist before finalizing an RFC.

- [ ] State the problem and motivation concretely.
- [ ] Define clear goals and non-goals.
- [ ] Describe current behavior with exact file references.
- [ ] Describe proposed behavior with enough detail to implement.
- [ ] Identify all affected crates and likely touch points.
- [ ] Include at least two alternatives with trade-offs.
- [ ] Explain backward compatibility and migration impact.
- [ ] Define test and validation work per affected crate.
- [ ] List key risks and mitigations.
- [ ] End with explicit open questions or decision points.
- [ ] Keep terminology consistent with Kirin docs and code.
